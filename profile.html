About me

Over the next 20 years we grew Gravitate into a full-service web agency providing strategy, design, ux, development, implementation, and hosting to a wide range of businesses and charities.

My role within Gravitate as Technical Director has been primarily focused on application analysis, architecture planning, software development, infrastructure, tooling, developer support, and security. Farmer Brown Golden Egg Hunt

The Farmer Brown Golden Egg Hunt was an AR experience developed in partnership with Work Communications to promote Farmer Brown eggs. The game made use of Niantic 8thwall to create a mobile egg hunt game. The game gave away $100k of prizes each year it ran.

The mobile game was an interactive egg hunt that presented the user with a 3d map of their surroundings and a mixed reality view to collect the eggs once found. People could then collect eggs to obtain prizes. Game application was responsible for tracking available and collected eggs, user onboarding, prize redemption, and reporting.

As the primary developer on this project, I was responsible for working with the client to define the game process, rules, and functionality. I then worked with our design team to wirefame the workflows and user interface. I was then responsible for the full implementation of the game and supporting systems.

Responsibilities:

  • Project scoping, feasibility tests, and documentation
  • Implement frontent gameplay in JavaScript with the Niantic 8thWall AR framework
  • Implement game login mechanism with Apple, Facebook, and Google login providers
  • Design database schema for storage of player, prize collectables, and claimed prize data
  • Geospatial functionality — mobile device location and orientation handling, Open Streetmap integration, and MySQL geospatial queries
  • Terraform infrastructure provisioning and management
  • Infrastructure specification and provisioning
  • Reporting, logging and monitoring with BetterStack, Raygun
  • Management reports for stakeholders including — Game play data, Prize draw and collection, geographical usage heatmaps

My Humm

My Humm is a venture between Ecrotek NZ and Awanui Scientific to provide easy-to-use and flexible testing of honey to apiarists, exporters, blenders, and packers of honey.

Myhumm is a React-based portal for honey producers to place orders for honey lab tests, view their test results, create custom reports, and run blending optimisations. It provides an API used by Awanui Scientific's laboratory systems to manage the placement of orders, update statuses, and deliver results. It also provides a honey blending optimiser service that takes a set of test results and optimises the blending of honey to meet a set of requirements.

As the lead developer and architect on this project, I was responsible for:

  • Scoping and planning with the client and their project management team
  • React user interface implementation
  • Data modelling and implementation
  • Symfony/PHP backend service development
  • Implementation of a provided coin-or/cbc optimization model using Python Azure Functions
  • Infrastructure design specification
  • GitHub Actions CI pipeline setup for container builds and deployment to Azure
  • Azure, Fastly CDN, and DNS management using Terraform
  • PDF generation of customised lab reports for exporters
